## 給你（執行者本人）的注意事項

### 法規 URL 驗證（你要做的人工工作）

`data/law_index.json` 範本中已有 8 部法規，但每筆的 `selaw_url` 可能是推估值，需要你實際驗證。執行步驟：

1. 開啟 https://www.selaw.com.tw/Chinese/RegulationNameQuery
2. 輸入法規名稱（例如「證券交易法」）
3. 點搜尋結果進入法規頁面
4. 從瀏覽器網址列複製真正的 LawID（URL 格式為 `LawArticle.aspx?LawID=XXX`）
5. 回到 `data/law_index.json` 修正對應條目的 `selaw_url`

剩下 35 部法規請依相同流程補完。Claude Code 寫好的 `validate_law_index.py` 可以幫你批次驗證 URL 是否有效。

### 部署順序建議

1. 先讓 Claude Code 完成 Phase 1，產出本地檔案結構
2. 你照 `01_GitHub_Pages_部署指南.md` 建立 repo、推 code、啟用 Pages
3. 拿到第一個可訪問網址後，繼續讓 Claude Code 做 Phase 2 之後
4. 每次 Claude Code 完成一階段，你 git push 一次，網址自動更新
5. 最後一階段做完後，加上自訂網域（如有需要）
